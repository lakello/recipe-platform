# Recipe Platform / RecipeMeal Planner

Production-like DevOps pet-проект advanced-уровня: платформа для публикации рецептов, планирования питания на неделю и автоматического формирования списка покупок.

Проект создаётся не только как приложение, но и как полноценная DevOps-платформа с контейнеризацией, Kubernetes, CI/CD, инфраструктурой в облаке, мониторингом, логированием, трассировкой, безопасностью и backup/restore.

## О проекте

**Recipe Platform** — сервис, где пользователи могут:

- искать рецепты;
- публиковать собственные рецепты;
- загружать фотографии блюд;
- лайкать и комментировать рецепты;
- подписываться на авторов;
- планировать меню на неделю;
- автоматически получать список покупок на основе плана питания;
- модерировать контент через административную панель.

Проект доступен через:

- Web-приложение;
- Android-приложение;
- Desktop-приложение для Windows;
- Desktop-приложение для Linux.

Основная цель проекта — отработать полный цикл разработки, доставки и эксплуатации production-like приложения с использованием современного DevOps-стека.

## Основные возможности

### Аутентификация и авторизация

Поддерживаемые способы входа:

- Email + password;
- Google OAuth;
- Яндекс OAuth.

Функции:

- регистрация пользователя;
- подтверждение email;
- вход по email и паролю;
- OAuth callback;
- выдача JWT access token;
- выдача refresh token;
- refresh token rotation;
- logout;
- хранение пользовательских сессий.

JWT передаётся в запросах через заголовок:

```http
Authorization: Bearer <token>
```

## Профиль пользователя

### Профиль содержит:

- имя;
- username;
- mail;
- аватар;
- описание;
- ссылки на соцсети;
- дату регистрации;
- количество подписчиков;
- количество подписок;
- количество опубликованных рецептов.

## Рецепты

### Пользователь может создавать рецепты с полями:


- название;
- описание;
- категория;
- ингредиенты;
- количество и единицы измерения;
- шаги приготовления;
- время приготовления;
- количество порций;
- калорийность;
- белки / жиры / углеводы;
- сложность;
- фотографии;
- теги;
- статус приватности.

### Статусы рецепта:


- Draft;
- Public;
- Private;
- Hidden;
- Deleted.
